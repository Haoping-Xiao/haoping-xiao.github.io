## 什么是 ReAct？

ReAct（**Re**asoning + **Act**ing）是 Yao 等人在 2022 年提出的框架，核心思想是让大语言模型在解决问题时交替进行**推理**和**行动**。

传统的方法要么只让模型「想」（Chain-of-Thought），要么只让模型「做」（直接生成动作）。ReAct 将两者结合，形成如下循环：

```
Thought → Action → Observation → Thought → Action → ...
```

## 为什么 ReAct 有效？

1. **推理引导行动**：模型先思考当前状态和目标，再决定下一步操作，减少盲目尝试
2. **观察修正推理**：每次行动后的观察结果会反馈给模型，帮助它调整策略
3. **可解释性强**：整个推理过程以自然语言呈现，便于调试和理解

## 一个简单的例子

假设 Agent 需要回答「法国的首都是哪里，那里现在几点？」：

```
Thought: 我需要先确认法国的首都是巴黎，然后查询巴黎的当前时间。
Action: search("法国首都")
Observation: 法国的首都是巴黎。
Thought: 现在我知道首都是巴黎，接下来查询巴黎的当前时间。
Action: get_time("Paris")
Observation: 巴黎当前时间是 14:30。
Thought: 我已经获得了所有需要的信息。
Action: finish("法国的首都是巴黎，当前时间是 14:30。")
```

## 实现要点

在实际工程中，ReAct Agent 通常需要以下组件：

| 组件 | 职责 |
|------|------|
| LLM | 生成 Thought 和 Action |
| Tool Registry | 管理可用的工具列表 |
| Action Parser | 从 LLM 输出中解析出结构化的 Action |
| Environment | 执行 Action 并返回 Observation |

## 常见陷阱

- **无限循环**：没有设置最大步数限制，Agent 可能反复执行相同操作
- **幻觉工具调用**：模型可能调用不存在的工具或传入错误参数
- **上下文膨胀**：多轮交互后 prompt 过长，需要在合适时机做摘要压缩
